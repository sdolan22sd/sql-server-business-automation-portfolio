# SQL-Based EDI 846 Inventory Automation

## Overview

This project modernized a legacy Microsoft Access-based EDI 846 inventory availability process by transitioning the solution to Microsoft SQL Server. The goal was to improve reliability, scalability, maintainability, and support customer-specific business rules while automating the generation of EDI inventory files.

---

## My Role

I assisted in designing, developing, testing, and maintaining the SQL Server components of the application. My responsibilities included developing stored procedures, views, and business logic, troubleshooting production issues, implementing customer-specific inventory rules, optimizing SQL performance, and supporting customer onboarding and production deployments.

---

## Technologies

- Microsoft SQL Server
- T-SQL
- Stored Procedures
- SQL Views
- User Defined Functions
- PowerShell
- SQL Server Agent
- EDI 846

---

## Business Problem

The previous inventory process relied on a legacy Microsoft Access application that had become increasingly difficult to maintain and scale. Customer-specific inventory rules were becoming more complex, making enhancements and troubleshooting challenging.

---

## Solution

The new SQL Server solution centralized inventory processing using stored procedures, views, and reusable business logic. Customer-specific inventory thresholds, supersede rules, pricing, and exclusions are applied automatically before inventory files are generated and distributed.

---

## Outcome

- Improved maintainability
- Reduced manual intervention
- Automated inventory processing
- Better support for customer-specific business rules
- Improved production reliability
